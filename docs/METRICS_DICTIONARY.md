# Словарь метрик

Все определения одинаково используются в Pandas notebook и BigQuery SQL.

| Метрика | Определение | Фильтр / знаменатель | Зачем нужна |
|---|---|---|---|
| GMV | Сумма `order_value` | `status != 'Cancelled'` | Оценка оборота после исключения отмен |
| Net Revenue Proxy | Сумма `order_value` | `status NOT IN ('Cancelled', 'Returned')` | Выручка, сохранившаяся после возвратов и отмен |
| Gross Profit Proxy | `Net Revenue - product_cost` | Только net-заказы | Прибыль до логистики, маркетинга и налогов |
| Gross Margin | `Gross Profit / Net Revenue` | Net Revenue > 0 | Сравнение эффективности выручки |
| Cancellation Rate | `Cancelled orders / all orders` | Все заказы | Контроль проблем до выполнения заказа |
| Return Rate | `Returned orders / non-cancelled orders` | Заказы без `Cancelled` | Контроль потерь после покупки |
| AOV | `Net Revenue / net orders` | Net-заказы с положительной выручкой | Средний успешный чек |
| Repeat Purchase Rate | Клиенты с 2+ net-заказами / все net-покупатели | Заказы без отмен и возвратов | Оценка способности продукта возвращать покупателей |
| Session Purchase Conversion | Сессии с `purchase` / все сессии | Уникальный `session_id` | Эффективность продуктовой воронки |
| M1 Retention | Покупатели, повторившие покупку через месяц / размер созревшей когорты | Когорта первой net-покупки | Раннее удержание покупателей |
| RFM Recency | Дни после последнего net-заказа | На дату snapshot | Давность последней покупки |
| RFM Frequency | Количество уникальных net-заказов | На клиента | Частота покупок |
| RFM Monetary | Сумма Net Revenue | На клиента | Денежная ценность клиента |
| Item Return Rate | Returned items / non-cancelled items | На категорию | Поиск товарных проблем |
| Aged Inventory Cost | Себестоимость непроданного товара старше 180 дней | `sold_at IS NULL` | Оценка замороженного капитала |
| Contribution per User | Средняя gross profit после субсидии | Все участники эксперимента | Экономический guardrail A/B-теста |

## Гранулярность

Главная аналитическая витрина `order_mart` имеет гранулярность:

> одна строка = один уникальный `order_id`

Перед объединением с заказами `order_items` агрегируются до `order_id`. После создания витрины выполняется проверка:

```text
COUNT(rows) = COUNT(DISTINCT order_id)
```

## Важные оговорки

- Net Revenue и Gross Profit являются proxy-метриками.
- В источнике нет налогов, маркетинговых затрат, стоимости поддержки и полной стоимости доставки.
- Retention отражает повторную покупку по календарным месяцам, а не активность в приложении.
- A/B-данные симулируются и не доказывают эффект в реальном бизнесе.
